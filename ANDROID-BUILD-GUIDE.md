# Building the 5-Gas Exhaust Analyzer Android App

This guide covers building the Android app using Bubblewrap CLI (Trusted Web Activity).

## Prerequisites

- **Node.js** (v18+ recommended) with npm
- **Bubblewrap CLI**: `npm install -g @bubblewrap/cli`
- **Java Development Kit** (JDK 17+)
- **Android SDK** (can be installed via Bubblewrap)
- Google Play Developer account ($25 one-time)

## Step 1: Deploy the Web App

1. Deploy the Flask app to PythonAnywhere (or any HTTPS-enabled host)
2. Ensure the following files are accessible via HTTPS:
   - `/` (main page)
   - `/static/manifest.json`
   - `/static/sw.js`
   - `/static/icons/icon-*.png` (all sizes)
   - `/templates/index.html`
3. Verify PWA compliance:
   - Open Chrome DevTools > Application
   - Check "Manifest" and "Service Workers" sections
   - Ensure `manifest.json` loads and service worker is active

## Step 2: Initialize Bubblewrap Project

From the `android-twa` directory (or any build dir), run:

```bash
bubblewrap init --manifest=https://YOUR-DOMAIN.com/manifest.json
```

Bubblewrap will:
- Download the manifest
- Extract app name, icons, colors
- Generate default Android project files

## Step 3: Configure the Project

Edit generated files as needed:

### `app/build.gradle.kts` (or `build.gradle`)
- Set `applicationId` (package name): e.g., `com.kricipepsi.exhaustanalyzer`
- Set `versionCode` and `versionName`
- Update `minSdkVersion` (>= 21 recommended)
- Target recent API level (>= 34)

### `app/src/main/AndroidManifest.xml`
- Verify permissions: `<uses-permission android:name="android.permission.INTERNET" />`
- Optional: add `<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />`
- Check `android:usesCleartextTraffic="false"` (should be false for HTTPS)

### `app/src/main/res/values/strings.xml`
- Confirm app name matches manifest
- Add any additional strings if needed

### Icons
- Replace generated icons in `app/src/main/res/mipmap-*` with your own (higher quality)
- At minimum provide: mdpi, hdpi, xhdpi, xxhdpi, xxxhdpi

### `app/src/main/java/.../MainActivity.kt`
- Usually no changes needed; TWA is handled by `androidx.browser.trusted.TrustedWebActivityIntentBuilder`

## Step 4: Build the App

### Debug build (for testing)
```bash
bubblewrap build --debug
```
Output: `app-debug.apk`

Install on device via ADB:
```bash
adb install -r app-debug.apk
```

### Release build (for Play Store)
```bash
bubblewrap build --release
```
Output: `app-release.aab` (Android App Bundle)

You'll need a keystore. Generate one if needed:
```bash
keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias
```

Bubblewrap will guide you through signing.

## Step 5: Google Play Store

1. Create developer account at https://play.google.com/console
2. Pay $25 one-time fee
3. Create new application:
   - Title: "5-Gas Exhaust Analyzer"
   - Description: Explain the diagnostic tool
   - Category: Tools > Automotive
   - Upload screenshots (from device running the app)
   - Upload icon (512x512), feature graphic, banner
4. Complete content rating questionnaire
5. Pricing: Free (with AdSense ads)
6. Submit for review (typically few hours to 1-2 days)

## Important Notes

- **AdSense**: Works seamlessly inside TWA WebView; no special handling required
- **Offline**: Service worker caches static assets; form submissions require network
- **Updates**: Deploying new web app version updates automatically (no app update needed)
- **Privacy**: App does not collect personal data. Include privacy policy linking to your site.
- **Permissions**: Only INTERNET is needed; avoid requesting device identifiers

## Troubleshooting

- **TWA not loading**: Check HTTPS and manifest accessibility. Use `adb logcat` to view errors.
- **Ads not showing**: Ensure AdSense script loads; test in regular Chrome first.
- **Service worker not registering**: Check browser console; must be served over HTTPS.
- **Build failures**: Ensure correct Java version and Android SDK paths.
